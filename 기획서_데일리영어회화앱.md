## 0. 핵심 방향 (v0.2 변경점)
- **AI 미사용**: AI 대화·AI 발음평가 없이, 사용자가 **직접 듣고 → 읽고 → 따라 말하는** 자가학습 방식으로 진행
- **개발 비용 제로 지향**: 유료 클라우드 API/서버 없이 **기기 내장 기능 + 앱 내 로컬 데이터 + 무료 도구**만으로 구현

## 1. 서비스 개요

### 1.1 한 줄 정의
**매일 한 문장(또는 한 다이얼로그)씩, 듣고 따라 말하며 부담 없이 이어가는 영어회화 습관 앱**

### 1.2 기획 배경
- 기존 어학 앱은 콘텐츠가 너무 많아 부담스럽거나, 단순 암기 위주로 "실제 말하기"가 빠져 있음
- "하루 1문장/1대화"라는 **작은 단위**로 진입 장벽을 낮추고 **습관 형성**에 집중
- 비용 부담 없는 구조(기기 내장 TTS·녹음, 로컬 데이터)로 출발 → 운영비 0에 가깝게 유지

### 1.3 핵심 가치 제안 (Value Proposition)
| 구분 | 내용 |
|------|------|
| **작게 시작** | 하루 1문장 또는 1대화 — "5분이면 끝" |
| **말하기 중심** | 듣기 → 따라 말하기(직접 발화) → 내 녹음과 비교 |
| **습관 형성** | 스트릭(연속 학습일), 알림, 가벼운 게이미피케이션 |
| **자가학습** | AI 없이 스크립트 기반으로 스스로 연습·점검 |
| **다국어 확장** | 영어 → 일본어 → 프랑스어… 동일 구조로 확장 |

## 2. 타겟 사용자

### 2.1 1차 타겟
- 영어 회화를 "시작은 했지만 꾸준히 못 하는" 20~40대 직장인/학생
- 학원·과외는 부담스럽고, 짧게 매일 하고 싶은 사람
- 혼자 부담 없이 소리 내어 연습하고 싶은 사람

### 2.2 사용자 시나리오 (예시)
> 출근길 알림을 받고 앱을 켠다 → 오늘의 문장 "Could you give me a hand?"를 듣는다(내장 TTS) → 화면의 문장·뜻·발음 기호를 보며 따라 말한다 → 내 목소리를 녹음해 원음과 비교해 본다 → 체크하면 스트릭 +1, 5분 완료.

## 3. 핵심 기능 (기능 명세)

### 3.1 기능 목록 & 단계 (영어 단독, 무비용)
| # | 기능 | 설명 | 단계 | 비용 |
|---|------|------|------|------|
| F1 | **오늘의 학습** | 매일 1문장 또는 1다이얼로그 제공 (난이도별) | 1차 | 무료(앱 내장 데이터) |
| F2 | **듣기(TTS)** | **기기 내장 TTS**로 문장 재생, 속도 조절 | 1차 | 무료(OS 제공) |
| F3 | **따라 말하기 + 녹음** | 듣고 따라 말하며 **기기 마이크로 녹음·재생** | **1차** | 무료(OS 제공) |
| F4 | **원음 비교 / 받아쓰기** | 내 녹음 ↔ 원음 나란히 비교 UI, 받아쓰기 자동채점 | **2차** | 무료 |
| F5 | **표현 학습 보조** | 문장/뜻/발음기호 + **정중한 표현 & 슬랭/네이티브 표현 병기** | 1차 | 무료(앱 내장 데이터) |
| F6 | **스트릭 & 진도** | 연속 학습일, 누적 학습량, 캘린더 | 1차 | 무료(로컬 저장) |
| F7 | **푸시 알림** | 사용자 지정 시간 리마인드 | 1차 | 무료(OS 로컬 알림) |
| F8 | **복습/북마크** | 지난 문장 다시 보기, 즐겨찾기 | 1차 | 무료(로컬 저장) |
| F9 | **1일 10단어 암기** | 매일 단어 10개, 뜻·예문, **플래시카드 + 퀴즈** | 1차 | 무료(앱 내장 데이터) |
| F10 | **로그인/계정** | 이메일·비밀번호 로그인(Supabase), 사용자별 진행도 분리·동기화, 게스트 학습 | 1차 | 무료(무료 티어) |

> ※ AI 대화·AI 발음 채점은 **제외**. "정답 발화를 듣고 따라 말하기 + 내 녹음과 비교"로 대체.
>
> **녹음/비교 단계 분리**
> - **1차(MVP)**: 듣기 → 따라 말하기 → **내 목소리 녹음·재생**(원음과 번갈아 들으며 스스로 점검)
> - **2차**: **원음-내녹음 나란히 비교 UI**(파형/속도 맞춤 재생), **받아쓰기 자동채점**으로 객관적 피드백 강화

### 3.2 2차 이후 (확장 — 여전히 무비용 우선)
- 받아쓰기(딕테이션): 들은 문장을 직접 타이핑하고 정답과 비교 (자동 채점, 비용 0)
- 셀프 체크리스트(자기 평가: 강세·억양·연결 등)
- 단어장/표현 노트 (로컬)
- 상황별 다이얼로그 팩(여행/카페/면접 등) 추가
- 위젯(홈 화면에서 오늘의 문장)
- 오프라인 완전 동작(데이터 앱 내장이므로 기본 지원)

## 5. 콘텐츠 구조

### 5.1 회화 데이터 모델 (다국어 확장 + 앱 내장 고려)
```
Language (en, ja, fr ...)
 └─ Level (beginner / intermediate / advanced)
     └─ Lesson (= 하루치)
         ├─ day, type: sentence | dialog
         ├─ situation (greeting, cafe, travel, work ...)
         ├─ target_expression
         ├─ sentences[]   { text, translation, phonetic }
         ├─ dialog_turns[] { speaker, text, translation }
         └─ native_variations[]   ← 정중한 표현 + 슬랭/현지 표현 병기
              { text, translation, register: "polite|casual|slang", note }
```
- **콘텐츠 베이스 = 미국 현지에서 자주 쓰는 표현(American everyday English)**
- 각 표현마다 **① 정중/표준 회화**와 **② 본토 사람들이 실제 자주 쓰는 캐주얼·슬랭 표현**을 함께 제공
  - 예: "How are you?" → (slang) "What's up? / How's it going?"
  - `register` 값으로 정중·캐주얼·슬랭 구분, `note`에 사용 상황·뉘앙스 설명
- 음성은 **별도 오디오 파일 없이 기기 내장 TTS로 실시간 재생** → 저장공간·비용 절감
- 콘텐츠는 **JSON 파일로 앱에 내장**(asset) → 서버 불필요, 오프라인 동작
- **언어 비종속 스키마** → 언어 추가 시 JSON 데이터만 추가, 코드 변경 최소화

### 5.2 단어 암기 데이터 모델 (1일 10단어)
```
VocabDay (= 하루치)
 └─ day
     └─ words[10] {
          word, pos(품사), meaning(뜻),
          example, example_translation,
          phonetic
        }
```
- 회화 콘텐츠와 **같은 day로 연동**(그날 회화에 나온 단어·관련 표현 위주)
- 학습 방식: 플래시카드(뒤집기) → 4지선다 퀴즈 → 완료 체크

### 5.3 콘텐츠 확보 방안 (무비용)
1. 미국 일상 회화에서 빈도 높은 표현을 직접 큐레이션하여 JSON 작성
2. 표준 표현 + 현지 캐주얼/슬랭 표현을 쌍으로 정리(자작 문장 위주, 저작권 무관)
3. 발음 기호는 표준 IPA 표기 활용
4. **초기 30일치 영어 회화 + 단어(300개) 제작 완료** → `/content` 폴더 참조

## 6. 기술 스택 (무비용 구성)

### 6.1 플랫폼 전략
> 요구사항: **Android 우선 → 이후 Android + iOS**, 다국어 확장

**권장: Flutter (크로스플랫폼, 무료 오픈소스)**

| 옵션 | 장점 | 단점 | 추천도 |
|------|------|------|--------|
| **Flutter** | 단일 코드로 Android+iOS, 다국어(i18n) 기본, 무료 | 일부 네이티브 플러그인 의존 | ★★★ (추천) |
| Kotlin(네이티브 Android) | 안드 최적화 | iOS 별도 개발 → 공수 2배 | ★★☆ |
| React Native | 웹 개발자 친화 | 음성/성능 이슈 가능 | ★★☆ |

→ iOS 확장 + 다국어 계획이면 **Flutter**가 비용·공수 모두 유리. Android 먼저 출시, 동일 코드로 iOS 빌드.

### 6.2 음성/기능 구성요소 (전부 무료)
| 기능 | 사용 기술 | 비용 |
|------|-----------|------|
| TTS(듣기) | `flutter_tts` (OS 내장 TTS 호출) | 0원 |
| 녹음(따라 말하기) | `record` / `flutter_sound` (기기 마이크) | 0원 |
| 녹음 재생 | `audioplayers` 등 | 0원 |
| 로컬 알림 | `flutter_local_notifications` | 0원 |
| (선택)받아쓰기 음성인식 | `speech_to_text` (OS 내장, 온디바이스) | 0원 |

> 모두 무료 오픈소스 플러그인 + OS 내장 기능. **유료 클라우드 API 없음.**

### 6.3 데이터/저장 (오프라인 우선 + 클라우드 동기화)
- 콘텐츠: **앱 내장 JSON(asset)** — 서버·DB 비용 0
- 사용자 진도/스트릭/북마크: **로컬 저장**(`shared_preferences` / `sqlite(drift)`)를 1차 저장소로 사용
- **로그인 시 클라우드 동기화**: 로컬 데이터를 Supabase에 백업·동기화 → 기기 교체/재설치 시 복구, 다기기 지원
- **오프라인 우선**: 비행기·지하철에서도 로그인 없이 학습 가능, 온라인 복귀 시 동기화

### 6.4 인증/계정 (Supabase)
- **Supabase Auth — 이메일/비밀번호 로그인**으로 시작 (비밀번호는 서버에서 해시 처리, 클라이언트 미저장)
- **구글 로그인은 추후 추가**, Apple 로그인은 iOS 출시 시 도입(심사 요건), Kakao는 국내 필요 시
- **게스트(비회원) 학습** 후 계정 연동 가능(이탈률 완화) — 로컬→클라우드 승계
- 사용자별 데이터 분리: 진행도/스트릭/녹음 메타가 사용자 계정에 귀속

#### 계정 찾기/복구
- **아이디(이메일) 찾기**: 본인 인증(휴대폰/이메일) 후 마스킹된 이메일 안내
- **비밀번호 재설정**: 가입 이메일로 **재설정 링크 발송**(Supabase `resetPasswordForEmail`)
- (데모는 백엔드가 없어 이름 기반 본인 확인 + 즉시 재설정으로 흐름만 시연)

#### 회원가입 수집 정보
| 구분 | 항목 | 용도 |
|------|------|------|
| 계정 | 이름(닉네임), 이메일, 비밀번호(+확인) | 로그인/식별 |
| 기본(통계) | **성별, 나이대**, 지역(선택) | 사용자 분포 분석·관리자 통계 |
| 학습 | 학습 목표, 현재 레벨 | 맞춤 추천·시작 난이도 |
| 유입 | 유입 경로(선택) | 마케팅 채널 분석 |
| 동의 | 이용약관(필수), 마케팅 수신(선택) | 법적 고지·알림 |

#### 사용자/진행 데이터 모델 (Supabase · Postgres + RLS)
```
profiles      { id(uuid = auth.uid), name, gender, age_group, region,
                goal, level, channel, marketing_opt_in, created_at }
user_progress { user_id, language, day, convo_done, vocab_done, completed_on(date) }
user_settings { user_id, voice_rate, notify_time, ... }
user_bookmarks{ user_id, language, day, type, ref }
```
- **RLS(Row Level Security)**: `user_id = auth.uid()` 정책으로 본인 데이터만 접근 → 사용자별 안전 분리
- 'N일 연속'은 `completed_on`(완료 날짜) 기준으로 **달력상 연속 일수**를 계산

#### 관리자 페이지(추후) — 통계 활용
- **성별·나이대·지역 분포**, 유입 경로별 가입수, 목표/레벨 분포
- 리텐션(D1/D7/D30), 평균 스트릭, Day별 이탈 지점, DAU/MAU
- 개인정보는 **집계(익명 통계)** 형태로만 노출, 개별 식별정보 접근은 권한 분리

#### 비용 (Supabase 무료 티어)
- 무료 티어: 인증 월간 활성 사용자(MAU) 약 5만, DB 500MB, 파일 1GB, API 무제한(공정사용) → **초기엔 사실상 $0**
- 주의: 무료 프로젝트는 **7일 무활동 시 일시정지**(접속 시 재개) — 초기 데모/소규모엔 무방
- 규모가 커지면 Pro($25/월~)로 전환

### 6.5 아키텍처 개념도 (오프라인 우선 + Supabase)
```
[Flutter 앱]
   ├─ 내장 콘텐츠 JSON (asset)        ← 서버 불필요
   ├─ OS 내장 TTS (듣기)
   ├─ 기기 마이크 녹음 → 로컬 저장 → 비교 재생
   ├─ 로컬 알림 (리마인드)
   └─ 로컬 DB (진도/스트릭/북마크)  ──sync──▶  [Supabase]
                                               ├─ Auth (이메일/비밀번호, 추후 Google)
                                               └─ Postgres + RLS (사용자별 진행도)
```

## 7. 수익 모델 (Monetization)
> 운영비가 거의 0이므로 무료로 시작해도 부담이 적음.

| 모델 | 내용 |
|------|------|
| **무료 + (선택)광고** | 기본 무료, 무료 사용자 대상 배너/보상형 광고로 소액 수익 |
| 콘텐츠 팩 판매 | 상황별 다이얼로그 팩을 인앱 구매(1회 결제)로 추가 판매 |
| 후원/도네이션 | "커피 한 잔" 후원(선택) |

> AI·서버를 쓰지 않으므로 **구독 모델은 후순위**. 초기엔 광고/콘텐츠 팩 정도로 충분.
